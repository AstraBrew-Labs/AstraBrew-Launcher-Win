## 界面
Tab页
第一Tab是 本地实例
第二Tab是 在线下载

## 版本切换

可以在本地和在线之间切换，也可以在本地实例之间切换。但只能切换一个版本作为当前版本，其他版本只能作为备选版本。

## 本地实例

一个列表，能让自动扫描 和 用户手动添加本地实例的酒馆实例。
一个平铺的列表，展示所有本地实例的版本信息，左边是版本号，左下是实例路径，右侧是功能按钮，包含查看信息、移除列表、切换版本。

## 在线下载
酒馆全部版本列表API：https://api.github.com/repos/SillyTavern/SillyTavern/releases
最新版本API：https://api.github.com/repos/SillyTavern/SillyTavern/releases/latest
一个可以下载最新版本的酒馆的面板，展示下载进度、状态、错误信息等。
面板里只显示最新版本的下载与安装按钮，其他版本是要让用户点击 安装其他版本按钮后弹出的弹窗，在弹窗里选择要安装的版本。
面板里的更新与安装都是覆盖安装，不会保留旧版本。
列表里显示出最新的5个版本，每个版本显示版本号、发布时间、大小、安装按钮。
在线下载只能安装一个版本，旧版本可以升级到新版本，也可以降级到旧版本，但是不能同时升级和降级。
安装按钮点击后，会开始下载对应版本的酒馆，下载完成后，会自动切换到该版本。
下载过程中，会展示下载进度、状态、错误信息等。

## 在线下载弹窗规则

用户点击下载与安装的时候，弹出下载与安装的弹窗，先判断Git和Nodejs是否存在，判断与设置里的保持一致，设置里设置的用内置的就调用内置的，是系统就调用系统的。

判断环境都没有或者缺少任意一个，都要直接终止下载和安装，并弹出警告，要提示用户安装git和nodejs，否则无法下载与安装。

判断环境都有后开始先调用Git 把最新仓库克隆到data/sillytavern目录

克隆完毕后，用NPM把依赖安装完毕。安装命令如下：
set NODE_ENV=production
call npm install --no-save --omit=dev --progress --foreground-scripts --verbose

下载与安装都要像命令行一样的输出详细进度与日志。
